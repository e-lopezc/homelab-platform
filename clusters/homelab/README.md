# clusters/homelab

Flux's entry point for the **homelab** cluster — the single path `flux bootstrap`
targets (`--path=clusters/homelab`). Everything the cluster runs is reconciled from here.

## Contents

| Path | What it is |
|---|---|
| `flux-system/` | **Generated by `flux bootstrap` — do not hand-edit.** The Flux controllers (`gotk-components.yaml`) plus the Git source and root sync (`gotk-sync.yaml`). |
| `platform.yaml` | Flux `Kustomization`s for the platform layer: `platform-controllers`, then `platform-configs`. |
| `apps.yaml` | Flux `Kustomization` for application workloads. |

## Reconcile order

These are Flux `Kustomization` objects (orchestration) — not kustomize overlays. They
chain with `dependsOn` so each layer is Ready before the next starts:

```text
flux-system              (root — reconciles this directory)
└─ platform-controllers  (./platform/controllers, wait: true)
   └─ platform-configs   (./platform/configs)
      └─ apps            (./apps, wait: true)
```

This ordering guarantees an operator's CRDs exist before the custom resources that use
them (e.g. cert-manager before its `ClusterIssuer`).

> The actual manifests live outside this directory, under `platform/` and `apps/`. This
> folder only holds the pointers that tell Flux **what** to reconcile and **in what order**.
